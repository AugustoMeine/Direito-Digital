### **1. Conceitos**

* **Calúnia e difamação:**
  * Calúnia é a ação de atribuir falsamente um crime a outra pessoa e difamar é a ação de atribuir um fato negativo que não seja crime a outra pessoa.
  * Um dos canais que possibilitam a calúnia e difamação seria as redes sociais. O anonimato e a viralização facilitam a disseminação de informações (podendo ser falsas).

* **Norma, pena e regulamentação:**
    1. Norma: Regras que orientam o comportamento online, estabelecendo direitos e deveres dos usuários e provedores.
    2. Penas: Sanções que podem variar desde multas até prisão, dependendo da gravidade da ação realizada.
    3. Regulamentação: Processo de estabelecer regras, normas e diretrizes para governar uma determinada atividade ou setor. Essas regras servem para garantir que as atividades sejam realizadas de forma segura, justa e eficiente, protegendo os interesses da sociedade e dos indivíduos.

* **Compliance digital:**
    * Conjunto de práticas e procedimentos que visam garantir a segurança da informação, a proteção dos dados pessoais e a conformidade com as leis e regulamentações.

## 2. Termos Técnicos

### 2.1 Engenharia Social

* A engenharia social é a interação com pessoas para que revelem/disponibilizem informações confidenciais ou realizem ações que beneficie quem está aplicando. 

* **Técnicas:**
    * **Phishing:** Enganar a vítima para que clique em links maliciosos ou forneça dados pessoais, através de meios eletronicos (e-mail falsos, mensagens falsas, etc).
    * **Pretexting:** Criar um cenário falso para obter informações confidenciais.
    * **Baiting:** Oferecer algo de valor, por exemplo um pen drive infectado, para que a vítima o pegue e infecte seu sistema/aparelho.
    * **Quid pro quo:** Oferecer algo em troca de informações, como um favor ou um serviço.
* **Possiveis Crimes:**
    * **Fraudes:** Obtenção de dados bancários, senhas e outras informações para realizar transações fraudulentas.
    * **Espionagem corporativa:** Obtenção de informações confidenciais de empresas para benefício próprio ou de concorrentes.
    * **Sextorsão:** Ameaça de divulgar conteúdo íntimo da vítima em troca de dinheiro.

### 2.2 Hacking

* **Tipos de hacking:**
    * **Hacking ético:** Realizado com autorização para identificar vulnerabilidades e melhorar a segurança de sistemas.
    * **Hacking malicioso (Crackers):** Realizado com a intenção de causar danos, roubar dados ou obter acesso não autorizado a sistemas.
    * **Hacktivismo:** Hacking motivado por causas políticas ou sociais.
* **Crimes relacionados ao hacking:**
    * **Invasão de sistemas:** Acesso não autorizado a sistemas computacionais.
    * **Danos a sistemas:** Alteração, destruição ou inutilização de dados ou sistemas.
    * **Furto de dados:** Roubo de informações confidenciais, como senhas e dados pessoais.

 ### 2.4 Licenças de Software

* **O que é e para que serve uma licença de software?**

Uma licença de software é um conjunto de termos e condições que define como um software pode ser utilizado, distribuído, modificado e copiado. Ela serve para proteger os direitos do autor, garantir que os usuários tenham permissões claras sobre o uso do software e estabelecer as limitações que o acompanham. Sem uma licença, o uso de um software pode ser restrito ou sujeito a questões legais, mesmo que o software seja gratuito. A licença também pode garantir que o software seja livre para a comunidade, caso seja um software de código aberto.

* **Tipos de Software**

#### Software Proprietário

Software proprietário é aquele cujo código-fonte não é acessível ao público. O desenvolvedor detém todos os direitos sobre o código e concede aos usuários uma licença para usar o software sob condições específicas, como limitações de uso e distribuição. Exemplos: Windows, Photoshop, Microsoft Office.

#### Software de Domínio Público

Software de domínio público é aquele cujos direitos autorais expiraram ou foram explicitamente renunciados pelo autor, permitindo que qualquer pessoa use, modifique e distribua o software sem restrições. O código-fonte é livre para qualquer uso sem necessidade de permissão.

#### Adware

Adware é um tipo de software que exibe anúncios enquanto o usuário o utiliza. Embora o software em si seja geralmente gratuito, ele gera receita para o desenvolvedor por meio de anúncios. O uso de adware pode ser um incômodo para o usuário, mas não necessariamente interfere na funcionalidade do software.

#### Freeware

Freeware é um software que é distribuído gratuitamente, mas com o código-fonte fechado. O usuário pode usá-lo sem pagar, mas não tem permissão para modificar ou redistribuir o código. Exemplos: Skype, Adobe Acrobat Reader.

#### Shareware

Shareware é um tipo de software que pode ser usado gratuitamente por um período limitado, mas depois exige pagamento para continuar o uso ou para acessar recursos adicionais. O objetivo do shareware é permitir que o usuário experimente o software antes de decidir se vai comprá-lo.

#### Software de Código Aberto (Open Source)

Software de código aberto é aquele cujo código-fonte é disponibilizado publicamente para que qualquer pessoa possa visualizá-lo, modificar e redistribuí-lo. Licenças de código aberto garantem que o software continue sendo livre, permitindo melhorias constantes pela comunidade. Exemplos: Linux, Apache, Mozilla Firefox.

#### O que é GNU General Public License (GPL)?

A GNU General Public License (GPL) é uma licença de software livre que garante que o software que a adota permaneça livre, tanto no uso quanto na distribuição. Ela permite que qualquer pessoa use, modifique e redistribua o software, desde que as modificações também sejam disponibilizadas sob a mesma licença. A GPL é uma das licenças mais populares para software de código aberto e foi criada pela Free Software Foundation (FSF) para promover a liberdade do software.

#### O que é GNU FDL, GNU LGPL, GNU AGPL, Licença Livre e Copyleft?

#### GNU FDL (GNU Free Documentation License)

A GNU FDL é uma licença criada para documentação de software. Ela permite que qualquer pessoa copie, modifique e distribua a documentação de forma livre, desde que mantenha a mesma licença e forneça acesso ao conteúdo original e modificado.

#### GNU LGPL (GNU Lesser General Public License)

A GNU LGPL é uma licença de software livre menos restritiva que a GPL. Ela permite que bibliotecas de software sejam usadas em projetos proprietários, desde que os desenvolvedores forneçam o código-fonte das modificações feitas à biblioteca. A LGPL foi criada para permitir maior flexibilidade no uso de bibliotecas em projetos comerciais.

#### GNU AGPL (GNU Affero General Public License)

A GNU AGPL é uma variação da GPL que foi criada para garantir que modificações no código, especialmente quando o software é utilizado em servidores, também sejam compartilhadas. Se um software licenciado sob a AGPL for usado em um servidor, os usuários devem ter acesso ao código-fonte, mesmo que não estejam executando o software localmente.

#### Licença Livre

Licença livre é qualquer licença que permita ao usuário usar, modificar, copiar e distribuir o software sem restrições significativas. A GNU GPL, a Apache License e a MIT License são exemplos de licenças livres, desde que o software seja utilizado de maneira conforme os termos estabelecidos.

#### Copyleft

Copyleft é um conceito baseado na ideia de "direitos autorais invertidos". Em vez de restringir o uso, copyleft exige que qualquer trabalho derivado de um software licenciado sob essa condição seja licenciado da mesma maneira. Ou seja, qualquer modificação ou redistribuição deve ser feita sob a mesma licença, garantindo que o software permaneça livre.

## Principais Licenças no GitHub

Quando um repositório é criado no GitHub, o usuário tem a opção de escolher uma licença para o seu projeto. Algumas das licenças mais comuns no GitHub incluem:

#### MIT License

A MIT License é uma licença permissiva que permite ao usuário fazer quase qualquer coisa com o software, como usá-lo, modificá-lo, copiá-lo e distribuí-lo, desde que a licença original e o aviso de direitos autorais sejam incluídos. É uma das licenças mais simples e populares.

#### Apache License 2.0

A Apache License 2.0 é uma licença permissiva que permite o uso, modificação e distribuição do software. Ela também fornece uma cláusula de patentes, que assegura que os colaboradores não possam processar os usuários do software por infração de patentes relacionadas ao código. É bastante usada em projetos corporativos.

#### GNU GPL v3

A GNU GPL v3, como mencionado anteriormente, garante que o software e suas modificações sejam distribuídos sob os mesmos termos de licença. A versão 3 inclui proteções contra práticas de "lock-in" e melhorias em relação à versão anterior, incluindo mais segurança sobre patentes e DRM.

#### BSD 2-Clause e BSD 3-Clause

As licenças BSD (Berkeley Software Distribution) são permissivas, permitindo o uso, modificação e redistribuição do software. A principal diferença entre a versão de 2 cláusulas e a versão de 3 cláusulas é a cláusula de não promoção, que impede que o nome do projeto ou do desenvolvedor seja usado para promover derivados sem permissão.

#### Creative Commons

Embora não seja amplamente usada para software, a licença Creative Commons é popular para conteúdo criativo, como texto, imagens, vídeos e música. Ela oferece diferentes formas de permissões e restrições em torno do uso e distribuição de conteúdo.

## Conclusão

Licenciar um software corretamente é fundamental para garantir que os direitos dos desenvolvedores sejam respeitados, além de definir as permissões de uso e distribuição do programa. As diferentes licenças oferecem uma variedade de opções para atender às necessidades específicas de desenvolvedores e usuários. A escolha da licença deve considerar o propósito do software, a colaboração com outros desenvolvedores e as permissões de uso que se deseja garantir.
## 3. Leis

### 3.1 Marco Civil da Internet - Lei nº 12.965/2014

* **Princípios fundamentais e sua aplicação em casos concretos:** O Marco Civil da Internet (MCI) é a lei brasileira que regula o uso da internet no país. Seus princípios fundamentais incluem a neutralidade da rede, a liberdade de expressão e o direito à privacidade. Na prática.

* **Aplicações:**
    * **Censura:** Define limites para a censura, garantindo a liberdade de expressão.
    * **Responsabilidade:** Define a responsabilidade dos provedores de internet em relação ao conteúdo gerado por seus usuários.
* **Responsabilidade dos provedores de internet:** Os provedores de conexão à internet e de aplicações de internet não serão tratados como responsáveis civis por danos causados por terceiros. No entanto, há exceções para casos como:
    * Quando o provedor tiver conhecimento da atividade ilícita e não tomar medidas.
    * Quando o provedor for o autor, coautor ou cooperador imediato da atividade ilícita.

### 3.2 LGPD (Lei Geral de Proteção de Dados) - Lei nº 13.709/2018

* A LGPD define dado pessoal como qualquer informação relacionada a pessoa natural identificada ou identificável. O tratamento de dados compreende todas as operações realizadas com esses dados, desde a coleta até a exclusão.
* **Princípios:** 
    * Finalidade: Os dados devem ser tratados para finalidades específicas, explícitas e legítimas.
    * Necessidade: A coleta e o tratamento de dados devem ser adequados, pertinentes e limitados ao necessário.
    * Qualidade: Os dados devem ser precisos, completos e atualizados.
    * Transparência: A empresa deve informar o titular dos dados sobre como seus dados serão tratados.
    * Controle do titular: O titular dos dados tem direito de acessar, corrigir e excluir seus dados.
* **Multas e sanções previstas:** As empresas que infringirem a LGPD estão sujeitas a multas de até 2% do seu faturamento no Brasil, podendo chegar a 50 milhões de reais.
